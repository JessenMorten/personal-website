Improve performance with the debounce function

JavaScript

March 4, 2022

Debouncing can limit the number of calls to a function, thereby improving the performance of your website.

The debounce function is a higher-order function that is very similar to the throttle function. The function will wrap another function which will only be invoked, when we're sure that we wan't to invoke it. It does this by delaying the actual function invokation, for a specified amount of time.

Usage example

A good situation to use a debounce function, is when implementing an instant search feature, which will search while the user is typing. The problem with such a feature is that it might put too much load on the backend, if we trigger a new search request everytime the user inputs a character. Instead, the debounced search function will not send the search request until the user is done typing. See how it works in the example below:

Debounced search example

Implementation

JavaScript// Elements
const searchStatusElement = document.getElementById("search-status");
const searchInputElement = document.getElementById("search-input");

// Debounces a function.
function debounce(func, milliseconds) {
    let timeout = null;
    return function (...arguments) {
        if (timeout) {
            clearTimeout(timeout);
        }
        timeout = setTimeout(() => func(...arguments), milliseconds);
    }
}

// Search function
function search(query) {
    if ((query || "").trim() === "") {
        searchStatusElement.textContent = "Type something in the input element below.";
    } else {
        searchStatusElement.textContent = `Searching for '${query}'`;
    }
}

// Wrap the search function with the debounce function
const debouncedSearch = debounce(search, 1000);

// Add event handler to search input
searchInputElement.oninput = event => {
    searchStatusElement.textContent = "Waiting until you're done typing...";
    debouncedSearch(event.target.value);
};
