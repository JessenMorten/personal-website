What are Higher-order functions?

Higher-order functions are functions that either takes one or more functions as arguments, or a function that returns another function. Functions that don't do any of these thing are called first-order functions, and you're probably using them all the time.

JavaScript// 'log' is a first-order function of the 'console' object,
// because it doesn't take a function as an argument or returns a function.
console.log("Hello World!");

In order to use higher-order functions you need to be working with a programming language that treats functions as first-class citizens. Functions in such a programming language are called first-class functions, and all it means is that functions can be passed as arguments and be returned as a result of another function. Higher-order functions are especially used in functional programming languages, such as Erlang, Haskell and F#.

Why should I use higher-order functions?

Prevent side-effects

A common goal in software development is to avoid side-effects, but this is actually quite difficult to do. Especially in object oriented programming it's very easy to introduce side effects inside instance methods, because we think that a method is probably only called once, so it won't matter... except when your colleage adds another call to the method one month later, and suddenly you start seing weird, unpredictable behavior in your application. I should add that you can cause side-effects inside higher-order functions if you really want to, but in general, higher-order functions tend to have less side-effects than instance methods with access to the internal state of the instance.

Re-usability

Since higher-order functions seem to have a more generic interface than instance methods, it is more likely that you can re-use the function somewhere else in your application. This will eventually save you development and unit testing time, it will result in less code that has to be maintained, debugged and fixed.

Readability

Using higher-order functions will also result in code that is more declarative than imperative. This is a topic for another post, but the most important thing about the declarative programming paradigm is that the code is easier to read because it expresses indend explicitly rather than implicitly. Put in another way, declarative code tells you what it is doing, and not how it is doing it. Consider the example below.

JavaScriptconst people = [
  { name: "John", age: 42 },
  { name: "Jane", age: 27 },
  { name: "Bob", age: 15 },
];

// Find teenagers, the imperative way
const imperativeTeens = [];
 
for (let i = 0; i < people.length; i++) {
const person = people[i];
  if (person.age >= 13 && person.age <= 19) {
    imperativeTeens.push(person);
  }
}

// Find teenagers, the declarative way
const declarativeTeens = people.filter(person => person.age >= 13 && person.age <= 19);

How to implement higher-order functions?

Taking a function as an argument

Let's look at how to implement some higher-order functions. We will start by implementing a function that takes another function as an argument. This higher-order function will measure the time it takes to execute the function passed as an argument, and print the result to the console.

JavaScript/**
 * Measures and logs the time it took to execute the function.
 * @param {function} func Function to execute.
 * @param {string} description Description of the function behavior.
 * @returns The returned result of the function.
 */
function measureExecutionTime(func, description) {
    console.time(description);
    const result = func();
    console.timeEnd(description);
    return result;
}
 
/**
 * Prompts the user for their name.
 * @returns The name of the user.
 */
function askUserForName() {
    return prompt("What's your name?");
}
 
// Measure how long it takes the user to enter their name
const answer = measureExecutionTime(askUserForName, "Ask user for name");
console.log("User's name is " + answer);
 
// OUTPUT:
// Ask user for name: 2757.75927734375 ms
// User's name is John Doe
Returning a function

Now it's time to return a function from another function. This higher-order function will take a math operator, represented by a string, and return a function that can carry out the math operation by calling it with two numbers.

JavaScript/**
 * Creates an operation based on the provided operator.
 * @param {string} operator The math operator.
 * @returns A function that can process the math operation.
 */
function createOperation(operator) {
    switch (operator) {
        case "+":
            return (a, b) => a + b;
        case "-":
            return (a, b) => a - b;
        case "*":
            return (a, b) => a * b;
        case "/":
            return (a, b) => a / b;
        default:
            throw new Error(`Invalid operator '${operator}'!`);
    }
}
 
const a = createOperation("+")(3, 3); // 6
const b = createOperation("-")(3, 3); // 0
const c = createOperation("*")(3, 3); // 9
const d = createOperation("/")(3, 3); // 1
const e = createOperation("%")(3, 3); // Error: Invalid operator '%'!
