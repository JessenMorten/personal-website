What is a pure function?

JavaScript

February 24, 2022

When is a function pure, and when is it impure?

A pure function is a function that will always return the same result, if it's called with the same arguments. Also, it should not have any side effects. Think of it like a mathematical function where the output is alway a direct result of processing the input. I think the best way to explain this, is to show two functions that are not pure.

Function that sometimes returns a different result

The function below is not pure. It returns true if the specified time is between 7am and 11am, except when the function is called on a Sunday, then it will always return false regardless of the time.

Javascriptfunction isStoreOpen(hour) {
    // If it's Sunday, the store is closed
    const sunday = new Date().getDay() === 0;
    if (sunday) {
        return false;
    }
 
    return hour >= 7 && hour < 11;
}

In order to convert this to a pure function, we could choose to also take the day as an argument. That would make it a pure function that will always return the same result, when called with the same arguments.

Function with side effects

It could be argued that the function below is the pure version of the function above, but it is not. It has a side effects because it also counts disappointed customers, and therefore it is not a pure function.

Javascriptlet disappointedCustomers = 0;

function isStoreOpen(hour, weekDay) {
    let isOpen = false;
 
    if (weekDay !== 0) {
        isOpen = hour >= 7 && hour < 11;
    }
 
    if (!isOpen) {
        disappointedCustomers++;
    }
 
    return isOpen;
}
